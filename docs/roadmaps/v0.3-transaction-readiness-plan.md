# Shiori v0.3 实施计划（交易可靠交付与发布就绪）

## 1. 目标

v0.3 目标是把当前“可观测、可回归”的交易链路升级为“可交付、可运营”的 beta 级能力：

1. 通知链路从“仅实时推送”升级到“实时 + 可补偿拉取”。
2. 订单链路从“支付闭环”升级到“状态机可运营闭环”。
3. 风险与治理能力落地（限流、重放防护、幂等一致性）。
4. CI 从“观察型”升级为“准入型”质量门禁。

## 1.1 执行状态（2026-03-04）

- [x] M1：通知可靠性（实时 + 补偿）
- [x] M2：订单状态机完备与审计
- [x] M3：网关治理与幂等一致性
- [x] M4：发布准入与文档收口

## 2. 范围

### In Scope

1. notify 事件持久化与拉取接口（按用户、按游标增量拉取）。
2. WS 重连补偿机制（支持 `lastEventId` 或等价游标语义）。
3. 订单状态机补全与可审计（新增状态迁移与审计记录）。
4. 网关级限流与重放防护（关键写接口）。
5. 统一幂等策略（下单/支付/取消等写接口）。
6. CI 增加“阻塞型 local-regression + 非阻塞 perf-stress”双层任务。
7. README/runbook 补齐“生产排障顺序 + 发布准入条件”。

### Out of Scope

1. 新营销能力（优惠券、活动、满减）。
2. 多区域/多活部署。
3. 离线推送（短信、邮件、站内信）完整体系。

## 3. 里程碑

## M1：通知可靠性（实时 + 补偿）

### 变更点

1. notify 增加事件持久化存储（按 `userId + eventId` 可检索）。
2. 新增拉取接口：按游标分页获取未消费事件。
3. WS 建连支持“从最近游标续传”的补偿语义。
4. 增加通知可靠性指标：补偿拉取命中、重放次数、事件滞后。

### 验收

1. WS 断连后 5 分钟内重连，客户端可通过补偿接口补齐丢失事件。
2. `OrderPaid` 事件在“实时成功”与“补偿拉取成功”两条路径均可达。
3. notify 指标可区分实时发送与补偿发送。

---

## M2：订单状态机完备与审计

### 变更点

1. 订单状态机从 `UNPAID/PAID/CANCELED` 扩展到可运营闭环状态（例如 `DELIVERING/FINISHED`）。
2. 增加卖家侧与管理员侧状态操作接口（带权限与状态校验）。
3. 增加状态迁移审计（操作者、来源、前后状态、原因）。
4. 增加状态迁移指标（按 `from -> to` 维度统计）。

### 验收

1. 用户、卖家、管理员三类角色的状态操作均有完整鉴权。
2. 关键非法迁移会被拒绝并返回稳定错误码。
3. 管理端可查询订单状态迁移审计。

---

## M3：网关治理与幂等一致性

### 变更点

1. 网关对关键写接口增加限流策略（登录、下单、支付）。
2. 强化签名时间窗与重放防护（超时/重放请求拒绝）。
3. 统一幂等键规则与错误码语义，补齐接口文档。
4. 增加治理指标：限流命中、签名失败、幂等冲突。

### 验收

1. 压测下不会因瞬时突发导致核心服务雪崩。
2. 重放请求可被稳定拦截，日志可审计。
3. 幂等冲突语义在各服务一致。

---

## M4：发布准入与文档收口

### 变更点

1. CI 将 `local-regression` 基线改为阻塞。
2. CI 保留 `perf-stress` 非阻塞趋势观察并产出归档。
3. README/runbook 增加“发布准入清单”：
   - 必过任务
   - 关键阈值
   - 回滚触发条件
4. 增加版本发布演练脚本（含失败回滚步骤）。

### 验收

1. 主干分支每次变更都经过阻塞型基线验证。
2. 性能趋势可追溯（至少保留近 7 次结果）。
3. 发布与回滚流程文档可在 30 分钟内按文档完成演练。

---

## 4. 风险与回滚

1. 事件持久化带来存储膨胀风险：需配置过期策略与分层保留。
2. 状态机扩展引入兼容风险：需保持旧接口返回语义稳定。
3. 限流策略误伤风险：先灰度观测再全量启用。

回滚策略：
1. 通知补偿功能通过配置开关可关闭，保留实时 WS 主路径。
2. 新状态迁移接口可单独回滚，不影响既有支付闭环。
3. 网关治理规则可按接口维度逐项回退。

---

## 5. 完成定义（DoD）

1. 交易通知支持“实时 + 补偿”双路径且可观测。
2. 订单状态机具备可运营闭环与完整审计。
3. 关键写接口具备限流、重放防护、幂等一致性。
4. CI 形成“阻塞 + 观察”双层基线体系。
5. 文档与脚本可支撑一次完整发布与回滚演练。
