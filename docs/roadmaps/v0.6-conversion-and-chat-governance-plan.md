# Shiori v0.6 实施计划（咨询转化、聊天治理与 RBAC 扩展）

## 1. 目标

v0.6 目标是把 v0.5 已落地的“未下单咨询聊天”升级为“可转化、可治理、可运营”的交易前链路：

1. 让咨询会话可直接引导到下单动作，缩短从咨询到成交的路径。
2. 增加聊天侧治理能力（拉黑、举报、违禁词、频控），降低骚扰与垃圾消息风险。
3. 扩展 RBAC 到“能力级权限”，可按用户封禁聊天等高风险功能。
4. 补齐聊天可观测与回归基线，保障多实例下的稳定性。

## 1.1 执行状态（2026-03-05）

- [ ] M1：咨询到下单转化闭环
- [ ] M2：会话治理与基础聊天治理能力
- [ ] M3：RBAC 权限扩展（能力级封禁）
- [ ] M4：聊天可靠性与可观测性增强
- [ ] M5：发布准入与文档收口

---

## 2. 范围

### In Scope

1. 会话内“商品卡片/下单入口”能力（买家可从聊天直达下单页）。
2. 交易状态自动播报卡片（下单/支付/发货/收货）在聊天会话内展示。
3. 会话级行为治理（拉黑、举报、违禁词、发送频控）。
4. 管理端基础治理面板（举报处理、拉黑状态查询、违禁词配置）。
5. RBAC 能力级权限扩展（用户可被封禁聊天/发布商品等能力）。
6. 聊天可靠性增强（补偿拉取、已读一致性、离线未读对齐）。
7. 聊天链路 smoke/perf 资产补齐并接入 CI 基线。

### Out of Scope

1. IM 完整社交能力（群聊、好友体系、朋友圈）。
2. 复杂智能客服/大模型自动回复。
3. 全站推荐算法与个性化排序系统。

## 3. 里程碑

## M1：咨询到下单转化闭环

### 变更点

1. 会话消息新增 `product_card` 类型，包含 `listingId/title/price/cover`。
2. 前端聊天窗口增加“去下单”快捷入口，支持带参跳转到目标商品。
3. 商品下单接口增加“会话来源标记”（如 `source=chat` + `conversationId`）。
4. 增加交易状态播报消息 `trade_status_card`：
   1. 下单后自动发送：“我已下单，正在等待支付”
   2. 支付后自动发送：“我已支付，请尽快发货”
   3. 发货后自动发送：“我已发货，请注意查收”
   4. 收货后自动发送：“我已确认收货，本次交易完成”
5. 增加转化指标：`chat_to_order_click_total`、`chat_to_order_submit_total`。
6. 增加状态卡片指标：`chat_trade_status_card_sent_total{status}`。

### 验收

1. 买家在会话中可一键进入下单流程并完成提交。
2. 后端可按会话维度统计咨询转化率。
3. 下单/支付/发货/收货四个关键节点均会在对应会话自动落一条状态卡片。
4. 非法会话或越权会话不能注入下单来源标记。

## M2：会话治理与基础聊天治理能力

### 变更点

1. 新增拉黑能力：
   1. `POST /api/chat/blocks/{targetUserId}`
   2. `DELETE /api/chat/blocks/{targetUserId}`
2. 新增举报能力：
   1. `POST /api/chat/reports`（会话/消息维度）
3. 新增违禁词治理：
   1. 管理端支持违禁词增删改查（精确词 + 关键词）
   2. 发送前命中检测，支持“拒绝发送”与“替换掩码”两种策略（可配置）
4. 发送频控（按用户 + 会话）：
   1. 短时间突发限制
   2. 触发后返回稳定错误码与剩余冷却时间
5. 管理端新增举报处理最小闭环（列表、处理状态、处理备注）。

### 验收

1. 拉黑后双方不能继续发送消息，但历史记录可查。
2. 举报可落库、可追踪处理状态。
3. 违禁词消息会按策略被拦截或替换，并有审计记录。
4. 高频刷消息会被稳定限流，且不影响正常用户。

---

## M3：RBAC 权限扩展（能力级封禁）

### 变更点

1. RBAC 从“接口角色权限”扩展到“用户能力权限”：
   1. 能力项示例：`CHAT_SEND`、`CHAT_READ`、`PRODUCT_PUBLISH`、`ORDER_CREATE`
2. 新增用户能力封禁接口（管理端）：
   1. `POST /api/v2/admin/users/{userId}/capability-bans`
   2. `DELETE /api/v2/admin/users/{userId}/capability-bans/{capability}`
3. 网关与服务层双重校验能力权限，避免绕过。
4. 能力封禁支持原因、操作人、开始时间、结束时间、审计记录。

### 验收

1. 被封禁 `CHAT_SEND` 的用户无法发送聊天消息，并获得稳定错误码。
2. 被封禁用户的其他未禁用能力不受影响（最小影响面）。
3. 管理端可查询封禁记录并可解封，操作全链路可审计。

---

## M4：聊天可靠性与可观测性增强

### 变更点

1. 补偿拉取接口：
   1. `GET /api/chat/conversations/{id}/messages?afterSeq=...`
2. 已读一致性增强：
   1. `read` 状态支持幂等更新与重放容错
3. 多实例观测指标补齐：
   1. 连接数、在线会话数、消息投递延迟、补偿命中率、限流命中率
4. perf 资产新增 `k6-chat-conversation.js`（建连、发消息、断线重连、补偿）。

### 验收

1. 断线重连后消息可通过补偿接口补齐，不重复、不丢失。
2. 多实例场景下消息投递成功率达到目标阈值（例如 >= 99.9%）。
3. 指标面板可区分实时路径与补偿路径。

## M5：发布准入与文档收口

### 变更点

1. CI 增加聊天治理回归任务（拉黑/举报/违禁词/频控关键路径）。
2. 发布清单新增聊天专项准入：
   1. 转化指标可采集
   2. 治理策略（违禁词/限流）可配置回退
   3. RBAC 能力封禁可生效且可回退
   4. 补偿链路 smoke 通过
3. README 增补 v0.6 能力说明与接口示例。
4. 新增 runbook：聊天治理故障排查与临时降级策略。

### 验收

1. v0.6 核心链路均具备自动化回归覆盖。
2. 发布演练可在 30 分钟内完成（含一次降级演练）。
3. 文档与实际接口一致，避免“实现与文档漂移”。

## 4. 风险与回滚

1. 风险：治理规则过严导致误伤正常会话。
   1. 处理：频控阈值配置化，先灰度后全量。
2. 风险：违禁词误判影响正常沟通体验。
   1. 处理：先“审计模式”观察命中，再切“拦截模式”。
3. 风险：RBAC 能力封禁配置错误导致用户能力异常。
   1. 处理：封禁操作双人复核 + 过期时间默认必填。
4. 风险：转化链路字段扩展造成下单接口兼容问题。
   1. 处理：新增字段全部可选，旧客户端可无感运行。
5. 风险：补偿拉取在高并发下增加数据库压力。
   1. 处理：按会话分页 + 游标索引优化，必要时加缓存层。

回滚策略：
1. 聊天转化入口可通过前端开关快速关闭。
2. 违禁词治理可切回“仅审计不拦截”。
3. RBAC 能力封禁可批量回滚到最近稳定策略快照。
4. 举报/拉黑功能可独立回退，不影响基础聊天收发。
5. 补偿拉取可降级为仅实时路径，保留核心可用性。

---

## 5. 完成定义（DoD）

1. 咨询会话到下单形成可观测的转化闭环。
2. 拉黑、举报、违禁词、频控四类聊天治理能力在线可用。
3. RBAC 能力级封禁（含聊天封禁）在线可用且可审计。
4. 聊天在断线重连与多实例场景下具备可验证可靠性。
5. CI、README、runbook 与 v0.6 实际能力一致。
